# curly-adventure

A text-based adventure game scaffolding written in TypeScript. No theme or real content yet — just generic placeholder rooms, items, and a working command parser.

Pre-production build running at [https://curly-adventure.netlify.app/](https://curly-adventure.netlify.app/)
[![Netlify Status](https://api.netlify.com/api/v1/badges/f6cbf486-1e89-4b58-ac83-d3e74fcc8f72/deploy-status)](https://app.netlify.com/projects/curly-adventure/deploys)

## Project structure

```
src/
  types/        Room and Item interfaces
  engine/       Parser (command → ParsedCommand) and GameState
  data/         Placeholder rooms and items
  main.ts       Browser entry point
  __tests__/    Unit tests
public/
  index.html    Game UI
  style.css     Terminal-style styles
  bundle.js     Generated by the build (git-ignored)
```

## Development

### Prerequisites

- Node.js 20+
- npm

### Install dependencies

```bash
npm install
```

### Run tests

```bash
npm test
```

### Type-check

```bash
npm run lint
```

### Build for the browser

```bash
npm run build
# outputs public/bundle.js
```

### Watch mode (rebuilds on save)

```bash
npm run dev
```

## Running locally with Docker

```bash
docker compose up --build
```

Open [http://localhost:8080](http://localhost:8080) in your browser.

Re-run with `--build` any time the `Dockerfile` changes or after updating npm dependencies.

### Live-reload during development

`public/bundle.js` is git-ignored. The `docker-compose.dev.yml` overlay mounts `./public` into the nginx container, so webpack output is served immediately without restarting Docker. The first webpack build will create `public/bundle.js` once you start the workflow below; nginx will begin serving it as soon as it exists — no prior local build required.

**Step 1** — start Docker with the dev overlay in one terminal:

```bash
docker compose -f docker-compose.yml -f docker-compose.dev.yml up
```

**Step 2** — in another terminal, run the dev auto-refresh workflow:

```bash
npm run dev:docker
```

This command runs both:

- webpack in watch mode (rebuilds `public/bundle.js` on save)
- BrowserSync proxy with automatic browser reload

Open [http://localhost:3000](http://localhost:3000) while developing. BrowserSync proxies the nginx container on `localhost:8080` and refreshes whenever files in `public/` change.

If you only want rebuilds (without auto-refresh), use:

```bash
npm run dev
```

For a one-off build:

```bash
npm run build
```

Use Docker `--build` only when the `Dockerfile` or dependencies change.

## Deploying to Netlify

Connect the repository to Netlify. The `netlify.toml` at the root configures:

- **Build command**: `npm run build`
- **Publish directory**: `public`

Netlify will run the build and serve the `public/` directory as a static site.

## Supported commands (in-game)

| Command | Shorthand | Description |
|---------|-----------|-------------|
| `look` | `l` | Describe the current room |
| `look at <item>` | | Look at a specific item |
| `go <direction>` | `n s e w u d` | Move in a direction |
| `take <item>` | `get`, `pick` | Pick up an item |
| `drop <item>` | `put` | Drop a carried item |
| `inventory` | `inv`, `i` | List carried items |
| `examine <item>` | `x`, `ex`, `inspect` | Examine an item closely |
| `use <item>` | | Use an item |
| `use <item> on <item>` | | Use one item on another item |
| `help` | `?` | Show command reference |
| `quit` | `exit`, `q` | Quit the game |
